## FAQ | 常见问题

**English:**

### General Questions | 一般问题

**Q: What is LuminaBridge?**

A: LuminaBridge is a next-generation high-performance AI gateway built with Rust that unifies access to 50+ LLM providers through a single OpenAI-compatible API.

**Q: Is LuminaBridge free to use?**

A: Yes, LuminaBridge is open-source and free to use under the Apache 2.0 license.

**Q: What programming languages are supported?**

A: LuminaBridge provides an OpenAI-compatible API, so you can use any language that supports the OpenAI API (Python, JavaScript, Go, Java, etc.).

**Q: How do I deploy LuminaBridge?**

A: You can deploy using Docker (recommended) or build from source. See our [Deployment Guide](https://luminabridge.github.io/docs/deployment).

**中文:**

### 一般问题

**问：什么是 LuminaBridge？**

答：LuminaBridge 是一个基于 Rust 构建的下一代高性能 AI 网关，通过单一的 OpenAI 兼容 API 统一访问 50+ LLM 提供商。

**问：LuminaBridge 免费使用吗？**

答：是的，LuminaBridge 是开源的，根据 Apache 2.0 许可证免费使用。

**问：支持哪些编程语言？**

答：LuminaBridge 提供 OpenAI 兼容 API，因此您可以使用任何支持 OpenAI API 的语言（Python、JavaScript、Go、Java 等）。

**问：如何部署 LuminaBridge？**

答：您可以使用 Docker（推荐）或从源代码构建。请参阅我们的 [部署指南](https://luminabridge.github.io/docs/deployment)。

### Technical Questions | 技术问题

**Q: What are the system requirements?**

A: 
- **Minimum**: 2 CPU cores, 4GB RAM, 10GB disk
- **Recommended**: 4+ CPU cores, 8GB+ RAM, SSD storage
- **OS**: Linux (Ubuntu 20.04+), macOS, Windows (WSL2 recommended)

**Q: Which database does LuminaBridge support?**

A: Currently, LuminaBridge supports PostgreSQL 14+. MySQL and SQLite support are planned for future releases.

**Q: How do I configure OAuth authentication?**

A: See our [OAuth Configuration Guide](https://luminabridge.github.io/docs/oauth). You'll need to register your application with the OAuth provider (GitHub, Discord, etc.).

**Q: Can I use LuminaBridge with my existing OpenAI integration?**

A: Yes! Simply change the `base_url` in your OpenAI client to point to your LuminaBridge instance.

**Q: How do I add a new AI provider?**

A: You can add new providers through configuration or by implementing a custom provider plugin. See our [Provider Guide](https://luminabridge.github.io/docs/providers).

**中文:**

### 技术问题

**问：系统要求是什么？**

答：
- **最低**：2 CPU 核心，4GB RAM，10GB 磁盘

- **操作系统**：Linux（Ubuntu 20.04+）、macOS、Windows（推荐 WSL2）

### Troubleshooting | 故障排除

**Q: LuminaBridge won't start. What should I do?**

A: 
1. Check the logs: `docker logs luminabridge` or check the log files
2. Verify your configuration file syntax
3. Ensure the database is accessible
4. Check that required ports are not in use

**Q: I'm getting authentication errors**

A:
1. Verify your API keys are correct
2. Check that OAuth providers are properly configured
3. Ensure tokens haven't expired
4. Review the authentication logs

**Q: Requests are timing out**

A:
1. Check network connectivity
2. Verify provider API status
3. Increase timeout settings in configuration
4. Check server resource usage

**中文:**

### 故障排除

**问：LuminaBridge 无法启动。我该怎么办？**

答：
1. 检查日志：`docker logs luminabridge` 或检查日志文件
2. 验证配置文件语法
3. 确保数据库可访问
4. 检查所需端口未被占用

**问：我遇到认证错误**

答：
1. 验证您的 API 密钥是否正确
2. 检查 OAuth 提供商是否正确配置
3. 确保令牌未过期
4. 审查认证日志

**问：请求超时**

答：
1. 检查网络连接
2. 验证提供商 API 状态
3. 在配置中增加超时设置
4. 检查服务器资源使用情况
